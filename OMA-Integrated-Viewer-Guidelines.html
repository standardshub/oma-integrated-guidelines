Add New User
  • The addition of a new user to the Integrated Viewer requires to input the following information:
    • Name
      • Name of the user.
    • Email
      • User email address.
    • Role
      • Admin
        • Administrator access to the application.
      • Staff
        • Full write access to all the functions in the application.
      • Officer
        • Partial write access to the functions in the application.
      • Editor
        • A more restrictive write access than the Officer to the functions in the application.
      • Members
        • Mainly read access only to the following tabs functions:
          • Work Program
            • Ability to create and delete own Change Requests.
            • Ability to share Change Requests among other members.
          • Change Request
            • Ability to Withdrawn or create a new revision of a document submitted by the user.
    • Password
      • Input by the staff at the time of creating the user account.
    • Password Confirmation

Your Change Requests

  • The left-hand side of the screen is a personal and private section. It contains Change Requests created by the end-user that haven't been submitted yet.
  • The Change Requests in this section of the screen are only visible to the account holder, they are NOT visible to the rest of user accounts. 
    • Note: Please, note that it is possible for an user to share her Change Request with specific members or with ALL the users of the Integrated Viewer.
Your Change Requests
Operation Description Problems
Create a CR
  1. Click in "CR" button to create a new Change Request.
    • The CR file name is pre-constructed at the time of creating the CR.
      • OMA-<WG>-<Year>-<incremental number>_<title>
  • Note: the Change Request title MUST be unique. The system doesn't accept two Change Request with the same name.
  
Edit a CR
  1. Click in the "Edit" button to start editing an existing, not submitted yet, Change Request.
  • Once the Editor opens it is possible to share with other users the Change Request by clicking in the "Collaboration" button at the top of the Editor Tool.
  • The options are:
    • Private (single person icon)
      • It is the default setting.
      • In this mode the CR is only visible by the creation until it is submitted to the Group.
    • Public (World icon)
      • The CR is made available to all users of the Integrated Viewer.
      • Public CRs can be seen when clicking in the "Edit" button.
    • Collaborate (collaboration icon)
      • The CR is made available to a selected group of end-users that have an account with the Integrated Viewer.
  • P1 - 23rd May problem after the latest changes.
  • The problem now is that if the document is marked as "Revision" after being submitted, then it is not possible at all to add any new comment.
Compare CRs
  1. Click the "Compare" drop menu to view all the commits (Changes) for a particular Change Request.
  2. Upon selecting a Change Request, a new window will open to compare the different changes apply to the Change Request.
  3. Select the oldest commit tick box followed by the newest commit.
  • Note: if you compare two commits following this sequence you will see the new additions of the newest Change Request highlighted in green and the deletions in red.

Note: Each commit contain the changes saved after click "Save" button in the editor.

  
Submit CRs
  • Click in "Submit" drop menu to select which of the existing, not submitted, Change Requests you want to submit to the Working Group.
  • Once the CR is submitted it will disappear from the private area of the user.
  
Delete CRs
  • Click in "Delete" drop menu to select which of the listed Change Requests you want to delete.
